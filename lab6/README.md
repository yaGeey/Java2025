# Лабораторна робота №6

Set, Двозв'язний список

## Теоретичні відомості

**Колекції в Java** - це об'єкти, які групують кілька елементів в одну одиницю. Колекції використовуються для зберігання, отримання, маніпулювання та передачі агрегованих даних. Java Collection Framework надає набір інтерфейсів та класів для роботи з групами об'єктів.

**Інтерфейс Set** - представляє колекцію, яка не містить дублікатів елементів. Моделює математичне поняття множини. Основні методи включають add, remove, contains для базових операцій, addAll, removeAll, retainAll для масових операцій, та toArray для конвертації в масив. Set не гарантує порядок елементів.

**Двозв'язний список (Doubly Linked List)** - це лінійна структура даних, в якій кожен елемент (вузол) містить три компоненти: дані, посилання на наступний вузол та посилання на попередній вузол. Переваги: ефективна вставка та видалення елементів з будь-якої позиції (O(1) якщо є посилання на вузол), можливість обходу в обох напрямках. Недоліки: додаткова пам'ять для зберігання посилань, повільний доступ до елемента за індексом (O(n)).

**Узагальнені типи (Generics)** - механізм мови Java, що дозволяє створювати класи, інтерфейси та методи, які можуть працювати з різними типами даних, забезпечуючи при цьому безпеку типів на етапі компіляції. Використання generics дозволяє виявити помилки типів на етапі компіляції замість runtime, уникнути необхідності явного приведення типів та створювати більш загальний та придатний для повторного використання код.

**Інтерфейс Iterator** - надає уніфікований спосіб обходу елементів колекції без розкриття її внутрішньої структури. Основні методи: hasNext() перевіряє наявність наступного елемента, next() повертає наступний елемент та переміщує курсор, remove() видаляє останній повернутий елемент. Iterator дозволяє безпечно видаляти елементи під час ітерації.

**Методи equals() та hashCode()** - використовуються для визначення рівності об'єктів та для роботи з хеш-базованими колекціями. Метод equals() порівнює вміст об'єктів, hashCode() повертає цілочисельне значення для об'єкта. Важливе правило: якщо два об'єкти рівні за equals(), вони повинні мати однаковий hashCode().
