# Лабараторна Робота №3 - Класи

## Інкапсуляція: `private` та гетери/сеттери

**Інкапсуляція** — це принцип приховування внутрішніх даних об'єкта та надання доступу до них через спеціальні методи. Це захищає дані від неправильних змін і робить код гнучкішим.

  * **`private`**: Модифікатор доступу, який робить поле або метод видимим лише всередині самого класу.
  * **Гетер (Getter)**: Публічний метод для отримання значення приватного поля. Зазвичай називається `getFieldName()`.
  * **Сеттер (Setter)**: Публічний метод для зміни значення приватного поля. Дозволяє додати логіку перевірки (валідацію). Називається `setFieldName()`.

Замість прямого доступу, який може зламати об'єкт, ми надаємо контрольований інтерфейс.

```java
public class Student {
    private int age; // Поле закрите від зовнішнього світу

    // Гетер: дозволяє лише прочитати значення
    public int getAge() {
        return this.age;
    }

    // Сеттер: дозволяє змінити значення, але з перевіркою
    public void setAge(int age) {
        if (age > 0) { // Валідація даних
            this.age = age;
        }
    }
}
```

## Перевизначення методів: `@Override`

**Перевизначення** — це надання власної реалізації методу, який вже існує в батьківському класі (всі класи в Java наслідують `Object`).

Анотація `@Override` це спеціальна позначка для компілятора та інших розробників.

  * **Для компілятора**: Він перевірить, чи дійсно такий метод існує в батьківському класі. Якщо ви зробите помилку в назві методу, компілятор повідомить про це.
  * **Для розробника**: Це робить код більш читабельним, одразу показуючи, що цей метод не новий, а перевизначений.

## Сортування: `Comparator`

**`Comparator`** — це інтерфейс, який дозволяє визначити власну, зовнішню логіку сортування для об'єктів будь-якого класу. Це дуже гнучко, бо не вимагає змін у самому класі.

Щоб використовувати Comparator для сортування масивів, потрібно імпортувати клас `Arrays`: `import java.util.Arrays`;

#### Створення складного компаратора (Java 8+)

Найзручніше створювати компаратори за допомогою статичних методів `Comparator`.

  * `Comparator.comparing(function)`: Сортує за одним полем за зростанням.
  * `Comparator.comparingInt(function)`: Оптимізована версія для `int` полів.
  * `.thenComparing(function)`: Додає вторинне сортування (якщо значення попереднього поля однакові).
  * `Comparator.reverseOrder()`: Використовується для сортування за спаданням.

## Посилання на методи: Оператор `::`

**Оператор `::`** — це скорочений синтаксис для лямбда-виразів, які просто викликають один метод. Робить код чистішим і більш читабельним.

Замість того, щоб писати лямбду, яка приймає аргумент і одразу передає його в метод, розробник просто посилаєтеся на сам метод.

**Приклад:**

  * **Довгий запис (лямбда):**
    `institution -> institution.getNumberOfStudents()`
    *(Беремо об'єкт `institution` і викликаємо його метод `getNumberOfStudents`)*

  * **Короткий запис (посилання на метод):**
    `EducationalInstitution::getNumberOfStudents`
    *(Просто кажемо: "використовуй метод `getNumberOfStudents` з класу `EducationalInstitution`")*

Обидва варіанти роблять те саме, але другий значно лаконічніший.
